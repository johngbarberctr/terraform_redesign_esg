# aci-apic — APIC fabric + VMM root (AEDCG + AEDCK)

Manages APIC-direct objects for both AFRICOM ACI fabrics (AEDCG and AEDCK)
in a single Terraform root. Uses two provider aliases and two `netascode/nac-aci`
module instances. Lab and production targets share this directory; environment
distinction is handled via tfvars files and GitLab CI variables.

## Data layout

```
data/
  nac-aci-shared/          tenant EUR, ESGs, vzAny contracts (same on both fabrics)
  nac-aci-aedcg/           AEDCG fabric-specific (fabric policies, access, VPCs)
  nac-aci-aedcg-prod/      AEDCG prod overrides (legacy VLAN pool, PhysDom, L3Dom, AAEP)
  nac-aci-aedcg-rendered/  VMM domain YAML — generated by scripts/render-vmm-yaml.sh
  nac-aci-aedck/           AEDCK fabric-specific
  nac-aci-aedck-prod/      AEDCK prod overrides
  nac-aci-aedck-rendered/  VMM domain YAML — generated by scripts/render-vmm-yaml.sh
```

`*-rendered/` directories are `.gitignore`d; regenerate before plan with:

```bash
make render
```

## Environment selection

| File | When |
|---|---|
| `lab.tfvars` | Lab APIC IPs, `manage_tenants = true` |
| `prod.tfvars` | Prod APIC IPs, `manage_tenants = false` |

`manage_tenants = true` lets this root own the `mso_tenant` resource (needed
in lab where tenant EUR may not yet exist). Prod sets it false because tenant
EUR is pre-created by the NDO NAC prod pipeline.

Local workflow:

```bash
# Lab
cp terraform.tfvars.example terraform.tfvars   # first time only
$EDITOR terraform.tfvars                        # set aedcg/aedck passwords, vCenter creds
make render
make plan                   # uses lab.tfvars by default (VARS_FILE ?= lab.tfvars)
make apply

# Prod (override var-file and pass prod secrets via env)
export TF_VAR_aedcg_apic_password='...'
export TF_VAR_aedck_apic_password='...'
export TF_VAR_aedcg_mcp_key='...'
export TF_VAR_aedck_mcp_key='...'
make plan    VARS_FILE=prod.tfvars
make apply   VARS_FILE=prod.tfvars
```

## GitLab CI

Four jobs in `aci-apic/.gitlab-ci.yml` (two pairs: lab + prod):

| Job | Stage | Trigger |
|---|---|---|
| `validate-aci-apic` | validate | MR or push to `aci-apic/**` |
| `plan-aci-apic` | plan | same |
| `plan-aci-apic-prod` | plan | MR or push to `aci-apic/**` |
| `apply-aci-apic` | deploy | manual, `main` only |
| `apply-aci-apic-prod` | deploy | manual, `main` only |

State keys (do NOT rename — live state exists):

- Lab: `aci-redesign`
- Prod: `aci-redesign-prod`

## Key variables

| Variable | Where set | Notes |
|---|---|---|
| `aedcg_apic_url` / `aedck_apic_url` | tfvars | APIC HTTPS endpoints |
| `aedcg_apic_username` / `aedck_apic_username` | tfvars | usually `admin` |
| `aedcg_apic_password` / `aedck_apic_password` | env / CI masked | never committed |
| `aedcg_mcp_key` / `aedck_mcp_key` | env / CI masked | >=8 chars, mixed |
| `vcenter_hostname_ip` / `vcenter_datacenter` / `vcenter_dvs_version` | tfvars | VMM domain prereqs |
| `vcenter_username` / `vcenter_password` | env / CI masked | |
| `manage_tenants` | tfvars | `true` = lab, `false` = prod |
