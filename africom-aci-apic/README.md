# africom-aci-apic — AFRICOM NIPR APIC-direct root (Kelley + Del Din)

Manages APIC-direct objects for both AFRICOM NIPR ACI fabrics (Kelley and Del Din)
in a single Terraform root. Uses two provider aliases and two `netascode/nac-aci`
module instances. Lab and production targets share this directory; environment
distinction is handled via tfvars files and GitLab CI variables.

## Data layout

```
data/
  nac-aci-shared/          AFR-DEL.Services tenant stub, ESG stubs (same on both fabrics)
  nac-aci-kelley/          Kelley fabric-specific (access policies, NADE02LF101/102)
  nac-aci-kelley-prod/     Kelley prod overrides
  nac-aci-kelley-rendered/ VMM domain YAML — generated by scripts/render-vmm-yaml.sh kelley
  nac-aci-deldin/          Del Din fabric-specific (access policies, NAIT03LF101/102/BL103/104)
  nac-aci-deldin-prod/     Del Din prod overrides
  nac-aci-deldin-rendered/ VMM domain YAML — generated by scripts/render-vmm-yaml.sh deldin
```

`*-rendered/` directories are `.gitignore`d; regenerate before plan with:

```bash
make render
```

## Environment selection

| File | When |
|---|---|
| `lab.tfvars` | Lab APIC IPs, `manage_tenants = true` |
| `prod.tfvars` | Prod APIC IPs, `manage_tenants = false` |

`manage_tenants = true` lets this root own the `mso_tenant` resource (use in
lab if AFR-DEL.Services needs to be created). Prod sets it false because the
tenant is pre-created by the NDO NAC pipeline.

**No Python venv required.** All `make` targets use bash scripts and `terraform`
only. Scripts call `python3` stdlib (`json`, `os`, `sys`) for JSON escaping —
no pip packages needed.

Local workflow:

```bash
# Lab
source scripts/set-apic-password.sh              # sets TF_VAR_kelley/deldin_apic_password
eval "$(./scripts/generate-mcp-key.sh kelley)"  # TF_VAR_kelley_mcp_key
eval "$(./scripts/generate-mcp-key.sh deldin)"  # TF_VAR_deldin_mcp_key
export TF_VAR_vcenter_hostname_ip='...'
export TF_VAR_vcenter_datacenter='...'
export TF_VAR_vcenter_username='...'
export TF_VAR_vcenter_password='...'
export TF_VAR_vcenter_dvs_version='unmanaged'
make init
make auth-check
make plan
make apply

# Prod
export TF_VAR_kelley_apic_password='...'
export TF_VAR_deldin_apic_password='...'
export TF_VAR_kelley_mcp_key='...'
export TF_VAR_deldin_mcp_key='...'
make plan    TFVARS_FILE=prod.tfvars
make apply
```

## GitLab CI

Six jobs in `africom-aci-apic/.gitlab-ci.yml` (three pairs: lab + prod):

| Job | Stage | Trigger |
|---|---|---|
| `phase0-validate-africom` | validate | MR, push to `africom-aci-apic/**` or `scripts/validate_fabric.py`, or `PROJECT=africom-apic` |
| `phase1-plan-africom` | plan | same (needs phase0 to pass) |
| `phase1-apply-africom` | deploy | **manual gate** — runs post-apply drift check |
| `phase0-validate-africom-prod` | validate | `PROJECT=africom-apic-prod` or `main` push |
| `phase1-plan-africom-prod` | plan | same |
| `phase1-apply-africom-prod` | deploy | **manual gate** |

State keys:

- Lab: `africom-aci-apic`
- Prod: `africom-aci-apic-prod`

## AFRICOM NIPR Implementation Plan — automation

> Full plan: `docs/AFRICOM/AFRICOM_Implementation_Plan.md`

This directory and `../scripts/validate_fabric.py` together automate the
automatable portions of the CX remediation plan. The table below maps each
plan section to its automation file.

### What is automated here

| Phase | Item | File |
|-------|------|------|
| **0** | Health check baseline (cluster, faults, EPG/BD/VRF, BGP peers, BGP prefixes, VMM) | `../scripts/validate_fabric.py` |
| **0.1** | APIC configuration snapshot (both sites) | `../scripts/validate_fabric.py --snapshot` |
| **0.2** | NDO backup trigger | `../scripts/validate_fabric.py --backup-ndo` |
| **0.6** | NDO schema export to JSON | `../scripts/validate_fabric.py --export-schema` |
| **1.1** | Rogue EP Control — Del Din | `data/nac-aci-deldin/phase1-deldin-settings.nac.yaml` |
| **1.2** | Port Tracking — Kelley | `data/nac-aci-kelley/phase1-kelley-settings.nac.yaml` |
| **1.3** | BFD on fabric-facing interfaces | `data/nac-aci-shared/phase1-fabric-settings.nac.yaml` |
| **1.5** | Disable Remote EP Learning | `data/nac-aci-shared/phase1-fabric-settings.nac.yaml` |

### What requires manual action

| Phase | Item | Why not automated |
|-------|------|-------------------|
| **1.4** | FISAC Permissive → Strict | No ACI provider resource — controller-level GUI only. Do one APIC node at a time. |
| **2** | VLAN pool separation, MCP, ILP consolidation | NAC YAML templates exist but require VLAN range audit first. |
| **3** | Firewall BGP fixes, next-hop-self, Fusion PBR | Firewall/router config — outside ACI tooling. |
| **4** | vCenter management migration, VMM service account | vCenter/ESXi/AD operations. |
| **5** | NDO template merge, VRF isolation | NDO GUI-only (`Move to Template` has no REST API in NDO 4.4.x). |
| **6.1** | ND standby node addition | VM deploy + cluster join — requires hypervisor access. |

### validate_fabric.py usage

```bash
# ── Full Phase 0 (recommended before every change window): ──────────────────
# Creates APIC snapshots, NDO backup, exports schema, saves health baseline.
python3 ../scripts/validate_fabric.py \
  --phase0 \
  --artifacts-dir ../scripts/baseline/pre-phase1 \
  --label pre-phase1

# ── Health check only (fast, read-only, safe any time): ─────────────────────
python3 ../scripts/validate_fabric.py \
  -o ../scripts/baseline/pre-phase1.json

# ── Post-change drift report: ────────────────────────────────────────────────
python3 ../scripts/validate_fabric.py \
  --compare ../scripts/baseline/pre-phase1/baseline.json \
  -o ../scripts/baseline/post-phase1/baseline.json

# ── Skip NDO (when ND is unreachable): ──────────────────────────────────────
python3 ../scripts/validate_fabric.py --skip-ndo
```

Credentials are read from environment variables:
`KELLEY_APIC_URL`, `KELLEY_APIC_USERNAME`, `KELLEY_APIC_PASSWORD`,
`DELDIN_APIC_URL`, `DELDIN_APIC_USERNAME`, `DELDIN_APIC_PASSWORD`,
`NDO_URL`, `NDO_USERNAME`, `NDO_PASSWORD`.

In CI, these are provided as GitLab masked variables. Set `PHASE0_FULL=true`
in the **Run Pipeline** dialog to activate the snapshot/backup/export actions
in the `phase0-validate-africom` job.
